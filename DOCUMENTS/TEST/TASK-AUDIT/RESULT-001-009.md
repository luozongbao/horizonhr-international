### Fix Applied
- **[FIXED]** `check.status` middleware was registered in `Kernel.php` but not applied to any route group. Suspended or deleted users with a valid Sanctum token could access `/api/student/*`, `/api/enterprise/*`, and `/api/admin/*`.
  - **Fix:** Added `check.status` to the main `auth:sanctum` middleware group in `routes/api.php`:
    ```php
    Route::middleware(['auth:sanctum', 'check.status'])->group(function () {
    ```
  - The logout/refresh/me endpoints under `auth:sanctum` (inside `prefix('auth')`) are **intentionally excluded** from this group — a suspended user must still be able to call logout to clear their token.
